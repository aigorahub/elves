# Plan: Elves v1.8.0 Run Control Hardening

## Mission

Harden Elves so unattended runs treat checkpoints, stop conditions, and paid compute correctly by
default. The release is done when the core skill files, reusable templates, durable docs, and
human-facing docs all describe the same `v1.8.0` operating model without the user having to infer
missing semantics from conversation history.

## Scope

### In Scope

- Make run control explicitly live metadata instead of planning-only guidance.
- Distinguish delivery checkpoints from true stop boundaries everywhere that matters.
- Add first-class visibility for paid compute, remote jobs, and other long-running resources.
- Update skill files, templates, durable docs, release notes, and human-facing docs together.

### Out of Scope

- New automation or orchestration code beyond small documentation scaffolding.
- Changes to the "you never merge" rule or the stage-then-launch model.
- New hosted services, MCP servers, or repo-specific tooling outside lightweight release hygiene.

## Non-Negotiables

- Update both `SKILL.md` and `AGENTS.md` when behavior changes.
- Keep the release lightweight and doc-focused; do not smuggle in unrelated tooling work.
- Preserve the stage-then-launch workflow and the "user merges" boundary.
- Treat documentation freshness as part of done, not cleanup.

## Test Strategy

- **Primary gate:** `python3 scripts/check_repo_consistency.py`
- **Secondary gate:** `git diff --check`
- **Review gate:** poll PR review comments and resolve any actionable drift before calling the docs synced

## Notes

- This release exists because an overnight run failed operationally even though the main skill
  theory was sound. The docs must make the correct operating model harder to miss.
